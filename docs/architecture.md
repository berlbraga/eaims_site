# Architecture

E-AIMS uses Next.js App Router with Server Components for read-heavy pages and Client Components only for OTP forms and video progress interaction.

## Main decisions

- Supabase Auth handles OTP sessions.
- Domain validation happens before OTP, in protected route checks, in the profile creation trigger and in `public.before_user_created_hook`.
- Supabase RLS is the primary data boundary.
- `student` and `admin` are the only roles. Professor is represented by `admin`.
- Cloudflare Stream is abstracted in `src/lib/cloudflare/stream.ts`.
- Materials use private Supabase Storage and signed URLs generated by server routes.
- Admin CRUD uses server actions and revalidates affected pages.

## Runtime boundaries

- Browser: public Supabase key only, OTP UI, video iframe.
- Server: service role bootstrap, protected route guards, material URL signing, Cloudflare token generation.
- Database: RLS, triggers, last-admin protection and domain enforcement.

## Security notes

The portal does not authenticate against any institutional provider. It verifies access to an authorized e-mail inbox only.
